# API Contract: SpotsEndpoint (Updated)

**Branch**: `002-ft4-ft2-modes` | **Date**: 2026-03-13

## Overview

The `SpotsEndpoint` is a Vaadin Hilla `@BrowserCallable` endpoint that provides band activity data to the frontend. This contract documents the changes needed to support multi-mode band activity.

## Methods

### getBandActivity()

Returns aggregated band activity for all active band+mode combinations.

**Signature**: `BandActivityResponse getBandActivity()`

**Response structure change**:

Before (current):
```json
{
  "bandActivities": {
    "20m": { "band": "20m", "mode": "FT8", "spotCount": 150, "score": 72, ... },
    "40m": { "band": "40m", "mode": "FT8", "spotCount": 89, "score": 65, ... }
  },
  "timestamp": "2026-03-13T12:00:00Z",
  "mqttConnected": true
}
```

After (this feature):
```json
{
  "bandActivities": {
    "20m_FT8": { "band": "20m", "mode": "FT8", "spotCount": 150, "score": 72, ... },
    "20m_FT4": { "band": "20m", "mode": "FT4", "spotCount": 23, "score": 58, ... },
    "40m_FT8": { "band": "40m", "mode": "FT8", "spotCount": 89, "score": 65, ... },
    "40m_FT2": { "band": "40m", "mode": "FT2", "spotCount": 3, "score": 45, ... }
  },
  "timestamp": "2026-03-13T12:00:00Z",
  "mqttConnected": true
}
```

**Key differences**:
- Map keys change from `"{band}"` to `"{band}_{mode}"` composite key
- Multiple entries per band (one per active mode)
- Only band+mode pairs with at least 1 spot in the aggregation window are included
- Scores include rarity multiplier effect (FT4 scores boosted ~1.5x on activity component, FT2 ~3x)

### getBandActivityForBand(String band)

Returns activity for a single band across all active modes.

**Signature**: `BandActivity getBandActivityForBand(String band)` → `List<BandActivity> getBandActivityForBand(String band)`

**Change**: Return type changes from single `BandActivity` to `List<BandActivity>` containing one entry per active mode on that band.

### getStatus()

No changes. Returns MQTT connection status and ingestion statistics.

## BandActivity DTO

No field changes. The `mode` field already exists and will now contain "FT4" or "FT2" in addition to "FT8". The `score` field reflects the rarity-adjusted score.

## Frontend Impact

The auto-generated TypeScript client (`SpotsEndpoint.ts`) will be regenerated by Hilla to reflect the updated return types. The frontend card system in `band-activity/index.tsx` already iterates all entries and matches by band+mode, so no card logic changes are needed -- only the mode registry needs FT4/FT2 enabled.
